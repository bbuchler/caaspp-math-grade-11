How This Pilot Works

Module 1 starts with Lesson 1 fully built so we can test the student experience, grading, work samples, and teacher follow-up before building the other 19 lessons. Lessons 2-20 are visible in the gradebook to show the course shape, but only Lesson 1 is active for this first review.

Lesson Structure

  1. Overview video and short reflection.
  2. Lesson vocabulary for the math terms students will use.
  3. Quick start question to check the main idea.
  4. Video questions, worked examples, infographic, and hands-on practice.
  5. Real-world problem with an image so the math has a concrete setting.
  6. Practice questions students can update as they learn.
  7. Exit check, finish line, optional bonus, and final lesson submit.

Grading Rules

  • Practice questions save as students go and can be updated.
  • Exit check questions are one attempt because they are the final evidence.
  • A lesson can still pass overall if the total score is 70% or higher.
  • A low exit check should still flag the student for teacher follow-up.
  • Teachers can open the work sample, override individual scores, and regrade individual AI responses.

AI Support

The Study Buddy should use the current lesson, visible page context, student answer, answer key, and rubric so it can help with the exact concept on screen. AI grading should give partial credit and feedback for written math explanations, while objective items still use deterministic checks when possible.

The teacher insight area should come from saved student responses. For now it flags students who need review. Once more students use the course, it can summarize trends, suggest small groups, and point to remediation lessons.

Teacher Flow

  1. Open the gradebook.
  2. Add a test student or teacher account.
  3. Click Lesson 1 in the gradebook to preview the lesson or open a student's work sample.
  4. Have the test student complete Lesson 1.
  5. Check the Lesson 1 breakdown, work sample, and AI insights for exit-check follow-up.

How This Scales

The gradebook is meant to handle a full class by showing one row per student and one column per lesson. With 20 lessons and 30 students, teachers should use the module tabs to narrow the view, then click a lesson score to open that student's work sample.

The detailed section breakdown should show only the selected active lesson, not all 20 lessons at once. For this pilot, the active detail panel is Lesson 1. After Lesson 1 is approved, each new lesson should reuse the same data structure so the dashboard can switch the detail panel to Lesson 2, Lesson 3, and so on.
